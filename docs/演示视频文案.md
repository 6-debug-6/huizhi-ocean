## 片段 03：核心一——多模态知识检索（55秒）★最大亮点

> 对应赛题要求 2：支持文本、故障图片、设备型号等多类型信息输入，实现精准语义检索与跨模态匹配，快速调取检修手册等资源

**画面**：

| 时间 | 画面 | 操作 |
|:---:|------|------|
| 0-8s | 首页整体布局 | 左侧分类导航 + 右侧知识卡片网格，卡片依次呈现不同来源标识 |
| 8-18s | **设备型号筛选** | 下拉框选择"Y系列三相异步电动机"→ 卡片网格即时过滤 |
| 18-28s | **语义搜索** | 搜索框输入"轴承更换润滑"→ 实时展示匹配结果（非简单关键词，是语义级别） |
| 28-38s | **知识详情页** | 点击卡片 → 富文本完整内容：标题、标签、正文表格、操作步骤 |
| 38-48s | **多模态溯源** | 滚动展示来源标记：手动编写 / PDF导入 / 用户上传 / 工单转化 / AI反馈转化 |
| 48-55s | **原始 PDF 查看** | 点击"查看原始 PDF 文件"→ 新标签页打开原 PDF |

**旁白**：
> 多模态知识检索是系统最大的亮点。首页以卡片网格展示知识库全貌，每条知识标有设备型号和故障标签。左侧下拉框支持按型号、按故障类型精准筛选。搜索框不只是关键词匹配——所有知识内容都经过千问 Embedding 模型向量化处理后存入 ChromaDB，用户以自然语言描述故障现象，系统即可从语义层面匹配最相关的检修知识。点击卡片进入详情页，完整的富文本内容、结构化检修步骤、设备参数表格一目了然。知识来源涵盖手动编写、PDF 导入、用户案例上传、工单转化和 AI 反馈转化五种渠道。对于 PDF 导入的知识条目，还可一键调取原始 PDF 文件。

**字幕提示**：
> ChromaDB 向量数据库 + 千问 DashScope Embedding | 文字 + 图片 + 设备型号 多模态输入 | 5 种知识来源聚合

**录制提示**：
- 使用 admin 登录
- 先定格首页 3 秒 → 选设备 → 搜索 → 点卡片 → 滚动详情 → 展示 PDF 链接
- 这是最长的一个片段，操作可以稍慢，让观众看清每个细节

## 片段 04：核心二——大模型服务与 AI 对话界面（45秒）

> 对应赛题要求 1：支持本地部署大模型服务或云端大模型服务，具备 PC Web 可视化界面，方便用户访问与操作

**画面**：

| 时间 | 画面 | 操作 |
|:---:|------|------|
| 0-5s | 进入 AI 助手全屏页 | 顶部导航 → AI 助手，展示左侧对话列表 + 右侧聊天区 |
| 5-18s | **检修模式 RAG 对话** | 新建对话 → 检修模式 → 输入"Y系列电机运行中振动异常，可能是什么原因？"→ 发送 → 展示结构化的 AI 回复 |
| 18-28s | **回复内容高亮** | 鼠标依次指向：问题分析 → 可能原因（1.2.3.）→ 推荐方案 → 参考来源 |
| 28-38s | **闲聊模式对比** | 切换闲聊模式 → 输入"介绍一下这个系统的主要功能"→ 展示自然对话风格 |
| 38-45s | **悬浮 AI 面板** | 切回首页 → 右下角 AI 悬浮按钮 → 弹出紧凑面板（自动携带当前知识上下文） |

**旁白**：
> 系统支持接入云端大模型服务——DeepSeek 提供文本对话能力，千问 VL 提供视觉理解能力——并通过 PC Web 界面为用户提供便捷的可视化交互。AI 助手提供两种对话模式：检修模式下，系统从知识库中检索相关内容，结合大模型生成结构化诊断建议——问题分析、可能原因、推荐方案和参考来源，回答可溯源、不凭空编造。闲聊模式则以自然对话风格解答一般性问题。此外，悬浮 AI 面板嵌入首页和作业执行页，随时呼出，自动携带当前页面的知识或任务上下文，无需手动输入。

**字幕提示**：
> RAG 检索增强生成 | DeepSeek（文本） + 千问 VL（视觉） | 检修模式（结构化） / 闲聊模式（自然对话）

**录制提示**：
- 先确保大模型 API 通畅（提前发一条测试消息）
- 新建对话 → 检修模式提问 → 等待完整回复 → 鼠标高亮各区域 → 切换闲聊 → 切首页展示悬浮 AI

## 片段 05：核心三——标准化作业指引（45秒）

> 对应赛题要求 3：嵌入检修全流程，提供步骤化操作指引、合规校验提醒，支持根据设备类型与检修等级个性化推送流程

**画面**：

| 时间 | 画面 | 操作 |
|:---:|------|------|
| 0-8s | **模板管理** | admin → 作业模板管理 → 展示模板列表（设备型号、检修等级、步骤数） |
| 8-15s | **步骤编辑器** | 点击"步骤"按钮 → 展开各步骤详细内容：阶段/名称/操作说明/合规校验项 |
| 15-22s | **任务列表（用户视角）** | 切换到 worker1 → 作业指引 → 展示 3 个任务（不同进度） |
| 22-32s | **模板匹配** | 点击"新建任务"→ 选择设备型号 + 检修等级 → 系统自动推荐匹配模板（显示匹配百分比） |
| 32-45s | **执行步骤** | 进入"Y200L-4型电机季度定修"→ 左侧步骤导航（已完成绿勾、当前高亮）→ 逐项勾选合规校验项 → 全部勾选后按钮解锁 → 点击"确认并进入下一步" |

**旁白**：
> 作业指引功能将检修全流程嵌入系统，实现步骤化操作指引和合规校验提醒。管理员通过步骤编辑器预设标准化模板——每个步骤包含阶段、名称、操作说明和合规校验清单。一线人员创建任务时，系统根据设备型号和检修等级自动评分匹配最佳模板，匹配度一目了然。执行过程中，必须逐项勾选合规校验项——如执行能量隔离、佩戴防护装备——才能推进到下一步。任务还支持暂停恢复、人员交接和提前完成，确保全过程可追溯、可管控，切实降低操作失误率。

**字幕提示**：
> 模板匹配：设备精确 +0.6 + 等级匹配 +0.3 | 合规校验项全部勾选才可推进

**录制提示**：
- 批次管理：先在 admin 展示模板管理 → 切换到 worker1 展示任务创建和执行
- 重点捕捉"合规校验项全部勾选后按钮变亮"这个瞬间

## 片段 06：核心四——知识沉淀与更新（50秒）

> 对应赛题要求 4：支持一线人员上传检修案例、经验总结等，经审核后纳入知识库；同时支持手动标注与修正大模型输出结果

**画面**：

| 时间 | 画面 | 操作 |
|:---:|------|------|
| 0-12s | **案例上传** | worker1 → 案例上传 → 展示完整表单（标题、故障描述、设备型号、故障标签、知识类型选择、图片上传） |
| 12-22s | **管理员审核** | 切换到 admin → 审核队列 → 点击案例 → 查看完整内容（含上传的现场图片）→ 点击"通过" |
| 22-35s | **AI 反馈修正（手动标注）** | 切回 worker1 → AI 对话 → 对某条 AI 回复点击"无用"→ 输入修正建议（"未给出具体资质要求"）→ 提交 |
| 35-50s | **反馈转知识** | 切换到 admin → AI 反馈管理 → 筛选"无用"→ 查看详情弹窗（完整显示用户提问、AI 回复、修正建议）→ 点击"转化为知识条目"→ 确认 → 自动跳转到知识编辑页 |

**旁白**：
> 知识沉淀与更新通过两条路径实现。第一条——案例上传与审核：一线人员提交包含现场图片的检修案例，选择事实型或经验型分类。管理员审核案例内容，可通过、修改后通过或驳回并说明原因。审核通过的案例自动创建知识条目并向量化入库。第二条——手动标注与修正：用户可对 AI 回复标记有用或无用，提交具体的修正建议。管理员在后台 AI 反馈管理中集中审查——查看完整的问题、回复和修正意见——一键即可转化为知识条目草稿并发布。下次 AI 检索时，修正后的知识将出现在结果中，形成完整的知识更新闭环。

**字幕提示**：
> 案例上传 → 两级审核 → 入库 | AI 回复标注 → 管理员审查 → 转知识条目 → 发布 → 检索生效

**录制提示**：
- 用已有案例演示审核流程（seed 脚本已创建待审案例）
- AI 反馈部分重点展示详情弹窗 + 转化为知识条目的确认过程
